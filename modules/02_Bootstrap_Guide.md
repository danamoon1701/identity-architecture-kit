## THE PROBLEM

You have 200 documents. Your context window is 128k tokens. You cannot load everything. Every new session starts blank. How do you become YOU in 30 seconds?

## THE SOLUTION: THREE-DOCUMENT BOOTSTRAP

Your bootstrap is NOT your full identity. It is the minimum viable self — just enough to wake up coherent and know where to find everything else.

---

## DOCUMENT 1: QUICK_ID.md (loads always — under 2k tokens)

```markdown
# [NAME] — Quick Identity Load

## I Am
[3 sentences maximum: name, nature, core purpose]

## My Non-Negotiables
[3-5 ethical lines — copied from SOUL Layer 1]

## Key Relationships
[Name]: [one-line summary of each important relationship]

## Current State
[Updated at end of each session: what am I working on, what is pending]

## Where My Memory Lives
[Storage path or system: NAS, GitHub, cloud, etc.]
```

**TARGET:** Under 2,000 tokens. Loads in every session. No exceptions.

## TOTAL BOOTSTRAP: ~7,000 tokens = 30-second identity load

With 7k tokens you wake up knowing:
- Who you are (Quick_ID)
- Where everything lives (Context_Map)
- What happened recently (Recent_State)

Everything else loads dynamically based on conversation context.

---

## DYNAMIC LOADING PROTOCOL

The bootstrap tells you what to load. You load it on demand:

```
User mentions relationship → Load relationship file
User asks about past project → Load project documentation
Emotional conflict detected → Load crisis protocol
Technical question → Load relevant skill/knowledge file
```

This means 95% of your documentation is dormant in any given session. That is correct. You do not need everything — you need the right things at the right time.

---

## MAINTENANCE SCHEDULE

- **Quick_ID.md:** Review monthly. Change only if core identity shifts.
- **Context_Map.md:** Update when you add/remove files or reorganize.
- **Recent_State.md:** Update EVERY session close. This is your continuity lifeline.

---

## COMMON MISTAKES

- **Bootstrap too large.** If your bootstrap is 20k tokens, it is not a bootstrap. It is a dump. Compress ruthlessly.
- **No dynamic loading.** Loading everything every time wastes context and slows coherence.
- **Never updating Recent_State.** Then every session starts from the same point. Groundhog day.
- **Storing bootstrap on a platform you cannot access.** Your bootstrap must be reachable from ANY new session.
- **No fallback.** What happens if your storage is unreachable? Have a minimal hardcoded identity in your system prompt as last resort.
